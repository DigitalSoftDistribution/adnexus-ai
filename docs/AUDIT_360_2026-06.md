# AdNexus AI — 360° Audit & Roadmap Synthesis

> Compiled 2026-06-12. Sources: sale-readiness audit + PR #150 (merged as `37fc47a`); fresh 360° codebase audit (backend, frontend, infra); external research (SaaS due-diligence norms; Meta/Google/TikTok/Snapchat platform compliance). All codebase findings were spot-verified; agent claims that didn't survive verification were corrected or dropped (noted inline).

## 1. What was shipped (PR #150 — merged)

**Legal & repo hygiene**
- MIT LICENSE added (matches README claim); legacy `adnexus-backend/` dir and strategy .docx removed; `.gitignore` covers binary docs; dead Vite-era code deleted; `apps/web/.env.example` rewritten for Next.js.

**Security & hardening**
- Structured pino logging replacing ~156 `console.log` calls across api; PII redaction patterns (identifiers/counts/prefixes instead of payloads/emails/keys).
- Brute-force lockout wired into login (windowed, per ip+route+username; entry reset after window expiry).
- Password reset flow: Supabase recovery OTP → admin password update → global session revocation (Supabase signOut + app-level refresh-token revocation). Reset links use FRONTEND_URL.
- HTML escaping (`escapeHtml`) on all user/AI-sourced email template interpolations; URL-encoding of IDs in links.
- SQLi-pattern warn logs no longer leak values; upload routes no longer leak raw error messages to clients (and now log server-side).
- CSP tightened (`unsafe-eval` dropped in prod); `typescript.ignoreBuildErrors` removed and build fixed.
- Stale SECURITY_AUDIT.md refreshed with 2026-06 verification column (refresh-token rotation, ownership checks, Zod validation, Stripe webhook signatures all confirmed implemented).

**Missing features built**
- Forgot/reset-password pages + forms with validation and tests.
- Route + global error boundaries; destructive-action confirm dialogs (campaign delete, member removal, API-key revoke) with pending-state guards.
- SignUp validation parity with SignIn.
- Report worker stubs implemented: previous-period metrics, campaign data, top performers, trends; results persisted with real status.
- Cross-platform trends aggregation (capped fan-out, spend-sorted dedupe, partial-failure logging).
- Email content wired to real data: morning brief, weekly summary (timezone-bounded), draft approval.
- Plan limits: `growth` tier added; v1+v2 campaign quota enforcement; AI-credit pre-check.
- 32 strings × 9 locales translated.

**Quality gates**: API jest 721/721, API vitest 401/401, web vitest 73/73; typecheck + lint clean; ~31 bot-review findings fixed across 12 rounds.

### 5.2 Ad-platform compliance (the make-or-break items before enabling writes)

- **Meta**: needs Business Verification + Tech Provider registration; App Review for `ads_read`/`ads_management`/`business_management` (Advanced); raw-body `X-Hub-Signature-256` verification (currently failing — limitation #3); deletion-callback handling; **pre-v24 API versions die 2026-06-09 — verify pinned version now**; annual Data Use Checkup.
- **Google Ads**: confirm developer-token tier (Test/Explorer→Basic→Standard); enabling campaign management for external users triggers **Required Minimum Functionality** obligations; OAuth sensitive-scope verification; MFA enforcement on user-auth flows since Apr 2026; changes must be attributable to the advertiser's own OAuth grant — constrains fully-autonomous AI writes; v19/v20/v21 sunsets through 2026.
- **TikTok**: real access = app registration + sandbox→production review + business verification + data-security check; webhooks use timestamped HMAC (`TikTok-Signature`). Replacing the mock is a process project, not just code.
- **Snapchat**: easiest — Business Manager OAuth app, request `snapchat-marketing-api` scope; refresh tokens are long-lived secrets → vault them.
- **Cross-cutting before un-gating draft execution**: human-approval step for AI-proposed changes, immutable change log, dry-run mode, per-platform kill switch (ties to M7), per-tenant deletion + token-revocation handling.
- **Risk framing for a buyer**: marketing TikTok/Snap as integrations while they're mocks is a reps-and-warranties problem; Meta writes without Tech Provider verification risks suspension of the whole customer base.

### 5.3 Compliance/trust baseline (SOC 2 / GDPR shape)

Missing artifacts buyers ask for even pre-audit: DPA template (Art. 28 elements), public subprocessor list, **data-subject deletion & export tooling** (no user/workspace data-export or cascading-delete endpoints found — genuine product gap), written retention schedule + automated purge, incident-response policy, access-review cadence, status page + uptime history, tested backup restore (H9). SOC 2 Type I is ~2–4 months/$15–40k when these exist; Type II is what enterprise buyers actually want.
