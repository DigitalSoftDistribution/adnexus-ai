## 1. What was shipped (PR #150 — merged)

**Legal & repo hygiene**
- MIT LICENSE added (matches README claim); legacy `adnexus-backend/` dir and strategy .docx removed; `.gitignore` covers binary docs; dead Vite-era code deleted; `apps/web/.env.example` rewritten for Next.js.

**Security & hardening**
- Structured pino logging replacing ~156 `console.log` calls across api; PII redaction patterns (identifiers/counts/prefixes instead of payloads/emails/keys).
- Brute-force lockout wired into login (windowed, per ip+route+username; entry reset after window expiry).
- Password reset flow: Supabase recovery OTP → admin password update → global session revocation (Supabase signOut + app-level refresh-token revocation). Reset links use FRONTEND_URL.
- HTML escaping (`escapeHtml`) on all user/AI-sourced email template interpolations; URL-encoding of IDs in links.
- SQLi-pattern warn logs no longer leak values; upload routes no longer leak raw error messages to clients (and now log server-side).
- CSP tightened (`unsafe-eval` dropped in prod); `typescript.ignoreBuildErrors` removed and build fixed.
- Stale SECURITY_AUDIT.md refreshed with 2026-06 verification column (refresh-token rotation, ownership checks, Zod validation, Stripe webhook signatures all confirmed implemented).

**Missing features built**
- Forgot/reset-password pages + forms with validation and tests.
- Route + global error boundaries; destructive-action confirm dialogs (campaign delete, member removal, API-key revoke) with pending-state guards.
- SignUp validation parity with SignIn.
- Report worker stubs implemented: previous-period metrics (inclusive-duration math), campaign data, top performers, trends; results persisted with real status.
- Cross-platform trends aggregation (capped fan-out 10 campaigns/platform, spend-sorted dedupe by platform:campaignId, partial-failure logging).
- Email content wired to real data: morning brief, weekly summary (timezone-bounded week windows, both workspace-scoping paths), draft approval.
- Plan limits: `growth` tier added; v1 campaign quota counts both scoping paths + pending create-drafts; v2 use-case enforcement; AI-credit pre-check.
- 32 strings × 9 locales translated (de, es, fr, it, ja, nl, pl, pt, ru).

**Quality gates**
- API jest 721/721, API vitest 401/401, web vitest 73/73; typecheck + lint clean monorepo-wide; ~31 review findings from Cursor Bugbot + CodeRabbit fixed across 12 rounds.

## 3. Deliberate review-skips (documented on PR, candidates for follow-up tickets)

- **Upload route error envelope** differs from global middleware (`{success,message,error}` vs `{success,error:{code,message}}`) — pre-existing contract; unifying = breaking change. Ticket: API error envelope unification (M).
- **Nonce-based CSP** (currently `unsafe-inline` for styles remains) — declared out of scope (M).
- **v1 quota check race** (concurrent creates can exceed quota by small margin) — benign at current scale; v2 re-enforces at approval (S to fix with tx/constraint).
- **`ReportParams.workspaceId` optional typing** — legitimate for ad-hoc reports; revisit when scheduled reports persist to DB (S).
- **Lifetime-spend heuristic** in trends campaign selection — bounded-fan-out tradeoff, documented in code (M to replace with windowed spend query).
- **CI workflow disabled on GitHub** — user action: Actions → CI → Enable workflow. All 4 jobs verified green locally. (XS, owner-side.)
